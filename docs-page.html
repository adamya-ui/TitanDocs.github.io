Development

This is docs for Titan React version. In this documentation you will find new features, components, composition functions, etc implemented by UI Team along with using Material UI and Custom Code.

Tip

You can find MUI component's docs at their official documentation

Let's install our project first, please see the installation instructions on how to install and run Titan.

Design Overview

Installation

Titan Admin is built using CRA. CRA is nicely documented to start off. You don't need to install or configure tools like Webpack or Babel. They are preconfigured and hidden so that you can focus on the code.

Warning

Please make sure you use node's LTS version which is recommended by official node site and not one with latest feature.

Create a New React App :

Create React App is a comfortable environment for learning React, and is the best way to start building a new single-page application in React.

It sets up your development environment so that you can use the latest JavaScript features, provides a nice developer experience, and optimizes your app for production. You’ll need to have Node >= 14.0.0 and npm >= 5.6

npx create-react-app my-app
cd my-app
npm start

Note

npx on the first line is not a typo — it’s a package runner tool that comes with npm 5.2+

Components

Components are independent and reusable bits of code. They serve the same purpose as JavaScript functions, but work in isolation and return HTML.Components come in two types, Class components and Function components, in this tutorial we will concentrate on Function components.

Navigation Menu

Design

This contains the main navigation system of the platform. It is designed to be located in the left of the screen. The menu can be expanded and collapsed.

  • Case 1: When the user is interacting with the tabular content or interactive area, the navigation menu collapses to allow more screen real-estate
  • Case 2: The sub-menu contents should also be visible in the collapsed menu in a hierarchal form. The sub-menu content should have left margin gap to show the hierarchical structure.
  • Case 3: All the content such as search, user profile, settings and notification should exist in the navigation panel to allow all the remaining space for other interactive content.
Actions Performed:
  • Main navigation method
  • Search
  • Profile switching
  • Settings
  • Notification
  • logo

Search

There are two search box that are consistent throughout the platform.

Top-Level Search
  • It will search the top-level user-flow as well as data (Data will vary according to user privileges)
  • It can show the recently searched term or frequently visited section in search suggestion
  • It will mostly be used to jump between different section of the user flow with manually going through the whole navigational flow.
Table-level search
  • This search will be limited to just the table.
  • All the search result will be limited to the table in the page.
  • Seach process can be combined with other options
  • Search can be streamlined by selecting the search section
Actions Performed:
  • Top-level navigation
  • Top-level searches
  • Table-level search

    Drawer

    This screen displays the details about one particular row. When one row of the table is clicked, a sidebar comes from the right to display the complete details and the table gets a little narrower retaining all the initial columns. The sidebar is vertically scrollable if the details exceed the screen. The sidebar can be closed by clicking on the close button of by clicking anywhere outside the sidebar.

    When the sidebar is active, both the sidebar along with the table should be interactive. When the other row from the selected row is clicked, new data is loaded in the sidebar without collapsing.

    Actions Performed:
  • Display all the description about the record in the sidebar
  • The sidebar should if vertically scrollable if the data in the sidebar is more.
  • The sidebar could be collapsed by clicking on the close button or clicking anywhere outside the sidebar
  • Clicking on another row other than the selected row, loads new data in the sidebar.
